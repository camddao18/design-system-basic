Getting Started
System Design là gì?
Quá trình định nghĩa kiến trúc, giao diện và dữ liệu cho một hệ thống thỏa mãn yêu cầu cụ thể. Đây là những quyết định đầu tiên — và khó sửa nhất — khi build sản phẩm.
🗺️
Tại sao System Design quan trọng?
Khi build một sản phẩm — kể cả website trading — những quyết định về kiến trúc cần được đưa ra từ sớm. Nếu chọn sai, về sau rất khó sửa mà không viết lại toàn bộ hệ thống.

System Design bao gồm tất cả các lớp: từ hạ tầng mạng → database → backend API → frontend → cách dữ liệu được lưu và truy xuất.

Câu hỏi cốt lõi: Nếu có 1 triệu người dùng cùng lúc, hệ thống có chịu được không? Nếu server sập, data có mất không? Nếu cần thêm tính năng, có phải viết lại không?
🌐
Infrastructure
Mạng, server, CDN, caching
🗄️
Data Layer
Database, storage, indexing
⚙️
Application
Backend, API, business logic
Áp dụng cho web trading
Trước khi code bất cứ thứ gì, cần quyết định: database nào lưu quiz + XP? Backend có cần server riêng không? Video/ảnh host ở đâu? Đây chính là System Design.
